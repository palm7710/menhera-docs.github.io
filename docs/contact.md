# 他者との連携

本人が説明できるときもあれば、説明の工程自体が負荷になるときもあります。  
このページは、必要以上に傷つけず、必要十分に伝えるための整理です。

## してほしいこと

- 要件を短く区切って伝える
- 急ぎかどうかを明示する
- `今すぐ決めなくてよい` 場面では、その旨を書く
- 返答の形式を指定する
- 休止や離席を拒絶と解釈しない

## してほしくないこと

- 気持ちを推測して断定する
- 返答が遅い理由をその場で問い詰める
- やさしい言い回しで複雑な依頼を包む
- 他人と比較して励ます
- 冗談で温度を上げる

## 伝えるときの書き方

読みやすさは、熱量より構造で決まります。  
以下の順序だと誤解が少なくなります。

1. 用件
2. 緊急度
3. 返答期限
4. 選択肢

```txt
用件: 明日の打ち合わせ開始時刻の確認
緊急度: 低い
期限: 今夜中で十分
選択肢: 10:00 / 10:30 のどちらかで返信してください
```

## 補足

配慮は、特別扱いではなく誤差を減らす手段です。  
相手を元気づけることより、余計な処理を増やさないことが有効な場面があります。
