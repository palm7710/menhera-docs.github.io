# トリガー整理

崩れ方にはきっかけがあります。
ここでは「弱さの証明」ではなく、再現しやすい条件としてトリガーを並べます。

## 予告のない変更

予定、言い回し、距離感が急に変わると、処理の前提が崩れて負荷が跳ね上がります。
特に「大したことではない変更」ほど周囲に共有されず、ひとりで取り残されやすくなります。

### 具体例

- 待ち合わせ時間が直前で変わる
- 会話の温度が急に冷える
- 合意したはずのルールが曖昧になる

## 返答が読めない沈黙

返信が遅いこと自体より、理由や見通しが分からないことが負荷になります。
沈黙は空白ではなく、最悪の想像が入り込む余地になります。

```yaml
trigger: unreadable silence
effect:
  - refresh message app repeatedly
  - lose focus on current task
  - prepare apology before any accusation exists
```

## 比較される場面

他人の安定や成果を見せられること自体が問題ではなく、自分の未整備さを突きつけられる文脈で負荷が増します。

### 先に決めておくと楽なこと

- 比較対象を見ない時間帯を作る
- SNS の通知を段階的に切る
- 元気な人の速度を標準値にしない

## 境界が曖昧な優しさ

やさしい言葉でも、責任範囲が不明確だと後から不安に変わります。
期待させる表現と実際の距離感に差があると、回復より混乱が残ります。
