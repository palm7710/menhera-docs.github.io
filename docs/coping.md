# Best Practices

安定化に効くのは派手な攻略法ではなく、地味な運用です。

## 推奨運用

- 定期的な連絡
- 明確な愛情表現
- 一貫した態度
- 返信できないときの事前連絡
- 試し行動を仕様にしない

## Anti-patterns

以下は事故率が高い運用です。

| Anti-pattern | 問題 |
| --- | --- |
| 放置 | `返信遅延` が重大イベントになる |
| 論理で否定する | 気持ちより正しさで押し切ると悪化しやすい |
| 他人と比較する | 自己否定や競争が起動する |
| 温度差を急に出す | `Silent Failure` へ入りやすい |

## 実務的なメモ

最終的に有効なのは次の三つです。

1. 不安定になる条件を共有する
2. 事前パッチを用意する
3. 再発時の連絡文を固定する
