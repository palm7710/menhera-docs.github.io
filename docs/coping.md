# 対処法

対処の目的は、元気に見せることではなく、処理量を下げて破綻を避けることです。  
効果のある方法は派手ではなく、たいてい小さく、繰り返し可能である必要があります。

## すぐに行うこと

- 入力源を一つずつ止める
- 返答を `今は保留` に固定する
- 次の予定を可能なら一件だけ残して、それ以外を遅らせる
- 水分、室温、姿勢など身体側の条件を先に整える

```txt
優先順位:
1. 音を減らす
2. 返答義務を減らす
3. 選択肢を減らす
4. 身体条件を戻す
```

## 連絡文のテンプレート

言葉を選ぶ余力がないときは、短い定型文を使います。  
誤解されにくいことを優先し、謝罪や事情説明は後回しで構いません。

```txt
反応が遅れています。内容は確認しています。
今は判断量を下げたいので、返答はあとでまとめます。
急ぎなら選択肢を二つまでに絞って送ってください。
```

## 避けたほうがよいこと

- 一度に全部を説明しようとする
- 平常運転を演じる
- 苦手な相手に確認役を任せる
- 夜間の自己分析を深掘りする

> 落ち着いてから考える、ではなく、考えなくても落ち着ける状態を先に作ります。

## 回復後の整備

再発防止は反省会ではありません。  
有効だった条件だけ残し、不要な自己評価を削ります。

```bash
# 事後メモの最小形
when="昨夜 22:10"
trigger="予定変更 + 通知音"
worked="無音 / 保留連絡 / 照明を落とす"
avoid="長文で説明しない"
```
