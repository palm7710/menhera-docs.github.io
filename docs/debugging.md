# Debugging

ここでいうデバッグは、責めることではありません。  
どこで状態が崩れたかを、ログから追う作業です。

## ログ分析

- 過去の会話履歴を確認する
- 不安が発生したポイントを特定する
- どの `trigger` が多いかを把握する

たとえば以下のように読みます。

```js
debug({
  trigger: '返信遅延',
  after: ['深読み', 'SNS監視', '長文']
})
```

## よくあるデバッグ観点

| 観点 | 見ること |
| --- | --- |
| 返信 | 遅延が説明なしで起きていないか |
| SNS | 返信より先に更新されていないか |
| 温度差 | 昨日までより急に短くなっていないか |
| 比較 | 過去ログ分析が始まる条件は何か |

## 予防パッチ

最も効くデバッグ結果は、次回用の `patch` に変えることです。

```js
kanojo.prePatch({
  trigger: '返信遅延',
  action: '事前連絡'
})
```
